# Étude : `rang` actuel vs `rangContrasted` (variante A — ADR-007)

> Produite le 2026-05-16 par `tools/rang-contrasted.ts`. Lecture seule, ne mute pas l'engine.

## Hypothèse testée

Le calcul actuel `rang = max(subStructures.rang) + 1` ne distingue pas :
- les sub-structures de **polarité opposée** (corrections — vraie imbrication fractale per protocole §5.3) ;
- les sub-structures de **même polarité** (extensions — raffinement de borne sans changement de niveau hiérarchique).

On définit ici une alternative :

```
rangContrasted(m) = 0 si pas de sub-structures
                  = max(sub.rangContrasted pour sub de polarité opposée) + 1
                  = max(sub.rangContrasted) sans incrément si tous les subs sont même polarité
```
