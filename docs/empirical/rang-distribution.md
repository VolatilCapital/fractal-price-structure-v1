# Étude empirique — Distribution des rangs

> Produite le 2026-05-16 par `tools/rang-distribution.ts`. Reproduisible via :
> ```bash
> pnpm exec tsx tools/rang-distribution.ts
> ```

## Objet

Quantifier la distribution des moves par rang sur chacune des deux fixtures, pour informer le choix d'un seuil de filtrage `minRang` (cf. ADR-003).

## Méthode

Pour chaque fixture :
1. Charger les bougies brutes.
2. Construire la structure via `FractalEngine.buildFromCandles` (deterministic, sans logger).
3. Pour chaque rang `r` rencontré, calculer :
   - nombre de moves à ce rang ;
   - répartition Growing / Reference / Archived ;
   - répartition Up / Down ;
   - durée moyenne d'un move (timeRange.end - timeRange.start) ;
   - nombre moyen et max de sub-structures descendantes (récursivement).
4. Afficher pour chaque seuil candidat `minRang ≥ N` combien de moves seraient conservés.

**Contrainte de cadrage** : aucune heuristique externe — uniquement les highs/lows et les cassures de structure produites par le protocole.

## Lecture

- La colonne **% du total** indique l'inflation produite par les racines de bas rang. Si la part du rang 0 est très élevée, c'est le bruit micro qui domine la sortie.
- La colonne **Durée moyenne** mesure la dispersion temporelle des structures par rang. On s'attend à voir cette durée croître monotonement avec le rang (un move de rang plus élevé absorbe par construction des sub-structures plus courtes).
- Le tableau **`minRang ≥ N`** est l'outil principal pour choisir un seuil par défaut côté visualizer / API : il dit combien de moves sont conservés et quel pourcentage de la sortie totale ils représentent.

## Décision

À discuter avec l'auteur — la valeur seuil dépend du compromis lisibilité/richesse souhaité. L'étude fournit les données, pas la décision.
